Part 1: Identifying Usability Problems

Picking a Web page

I picked this web page because I think it is “simple yet poorly-designed”: It is a simple website for a family-owned pizzeria, which basically shows its location, menu, introduction, and pictures.

Finding Problems

Mental vs Conceptual model

Mental model:

When entering the website of a pizzeria, people would mostly think of :

  1. See the menu
  2. How to order (phone or online?)
  3. Check the pizzeria's info (location, phone, hours)
Conceptual model:

This website provides the conceptual model in these parts:

  1. Showing the menu
  2. Providing the pizzeria's info (location, phone, hours)
  3. Showing the pictures
  4. Introducing the history(about us)

Usability

  1. Menu- hard to spot on large devices.

    2. One of the most important contents of the website is the menu. On the mobile side, the menu is standing out at the center, with the high-contrast comparison for the background and the black border.

    However, the menu button is not very visible on larger devices, for example, it is in the middle of the left column, where the eye-catching middle is a large picture of the interior.

  2. Hard to spot important information

    3. Similiarly, when the user visits the website, another important thing is the contact information for the pizzeria. In this case, it would be the location, phone number, hours, and delivery availability. As discussed above, the website's large device display did not provide a proper text hierarchy for the information. They are located in the top right corner, with a relatively small font. Such an arrangement will make it difficult for the users to find the information effectively.

  3. Low font color contrast
    4.

    Some of the texts are in low contrast with the background, which makes them hard to read. For example, the navigation “Home, Gallery, About Us” bar, when selected, turns to a grey color, which is really hard to read on the orange background. The same with the “show more” at the bottom of the gallery.

  4. Misaligned paragraph
    5.

    The whole paragraph in the “About Us” section is awkwardly arranged. The paragraph is too compact, and there are unnecessary line breaks, which makes the reading experience a bit difficult.

  5. Gallery pictures do not have proper labels
    6.

    The pictures in the “Gallery” are named with numbers and do not have any comments on them, and the customers will have no idea what the dish is called.

  6. Empty white region
    7.

    The website is scrollable, on both mobile and desktop devices. However, on the desktop device with a larger display, there is a blank space with the copyright information, which is not part of the footer. There might be something wrong with the structure of the code, and I will try to figure it out in a later part.

  7. Misplaced facebook link
    8.

    The website provides the function of facebook likes. Its placement looks fine on the larger device display, but the button is misplaced in the mobile version, which is misaligned with the “find us on facebook”.

Learnability

As mentioned in the previous evaluation of usability, the first-time visitor who wants to see the information and the menu would have to browse between the left and right of the website. Since this website does not provide online order service, its function is fairly simple and straightforward. The users would not spend too much time on getting their target, but it still would take more time than expected.

Memorability

Similarly, since the website's function is fairly simple, the user would not have too much trouble getting their target when revisiting the website after a while.

Accessibility

Results from WebAIM WAV:

  1. No headings.

    2. There is no headings for the website, made it poorly structured.

  2. Text color contrast too low.
    3.

    The color contrast between the text and the background is too low, making it hard to read.

  3. Text too small.
    4.

    Some text are too small, also making it very hard to read.

  4. “Menu” link to PDF might cause accessibility problems.
    5.

    The menu is linked to an external PDF file. I am not sure how to fix this though...

  5. Alternative text not used properly
    6.

    The alt description is not properly used. It might be better to add more descriptive texts.

Annotations:

  1. For the container of the “menu” background, I use the position: relative; so that i would keep a relative position with the button.

  2. I also use display: block for each div because there are not many flex elements required (besides the gallery and the later info images, which I use the row and column to adjust them better accordingly).

  3. For h1,h2 and h3, I am using rem font size so that they would be relative to the root element, which would be responsive in different device sizes while keeping the proper text hierarchy.

  4. For side-by-side elements, such as the image and text in location & hours, I used the float:left and right. I also adjusted the margin for a better appearance.

  5. I adjusted the image sizes so that they will align better in the gallery. For the gallery layout, I referred to w3schools for the alignment and display.

  6. There are interactive elements on the page, though not many.

    • When hovering above the menu button, its color changes and implies that it is clickable; when clicked, it links to the menu pdf file. I know it is not a good option, but since time is limited and I do not know much about interactive page construction, I will leave it as it was. Sorry.

    • When scrolling down, the “top” button appears on the right side. I used a little Javascript here (credit to w3schools.)

    • I have also linked the Facebook like button at the bottom, which is clickable. I get the code from Facebook.

    Many thanks to w3schools! I would not finish it without its amazing tutorials :')
